# Geometric_lib - Документация по тестированию

## 1. Цели и задачи тестирования

### Основные цели
- Проверить корректность вычисления площади и периметра геометрических фигур
- Подтвердить соответствие реализованных функций математическим формулам

### Задачи тестирования
- Проверить вычисление площади для всех типов фигур
- Проверить вычисление периметра для всех типов фигур
- Убедиться в правильной обработке различных типов входных данных

## 2. Описание продукта

### Функциональность
Продукт представляет собой библиотеку для вычисления площади и периметра четырех геометрических фигур:
-  Треугольник
-  Круг
- Квадрат
- Прямоугольник

### Особенности
- Поддержка целочисленных и дробных значений
- Стандартные математические вычисления

### Требования
- Корректность математических вычислений
- Обработка различных числовых форматов
- Соответствие стандартным математическим формулам

## 3. Область тестирования

### Тестируемые модули
| Модуль | Функции | Описание |
|--------|---------|----------|
| `triangle.py` | `area()`, `perimeter()` | Вычисления для треугольника |
| `square.py` | `area()`, `perimeter()` | Вычисления для квадрата |
| `rectangle.py` | `area()`, `perimeter()` | Вычисления для прямоугольника |
| `circle.py` | `area()`, `perimeter()` | Вычисления для круга |

## 4. Стратегия тестирования

### Методы тестирования
- **Модульное тестирование**: тестирование отдельных функций
- **Функциональное тестирование**: проверка соответствия функциональным требованиям

### Техники тестирования
- **Модульное тестирование** - проверка отдельных функций вычисления площади и периметра
- **Классы эквивалентности** - группировка входных данных (целые, дробные, большие числа)

### Типы тестирования
- **Функциональное тестирование** - проверка корректности математических расчетов
- **Тестирование точности** - валидация вычислений с плавающей точкой

## 5. Критерии приемки

### Условия успешного завершения тестирования
-  Все модульные тесты выполняются без ошибок
-  Коэффициент покрытия тестами ≥ 90%
- Корректная обработка целочисленных и дробных значений
-  Соответствие результатов математическим формулам

### Критерии качества
- **Точность вычислений**: отклонение ≤ 0.0001 для дробных чисел
- **Полнота тестирования**: все функции протестированы

## 6. Ожидаемые результаты

### Отчеты о тестировании
- **Статус тестирования**: пройдено/не пройдено для каждого теста
- **Метрики покрытия**: процент покрытия кода тестами
- **Статистика выполнения**: количество успешных/неуспешных тестов

### Метрики качества
- **Процент успешных тестов**: ≥ 90%
- **Количество тестовых случаев**: 32 теста (по 4 на каждую функцию)

## Запуск тестов

- **Тестовый метод =** функция, начинающаяся с test_

- **Тестирование =** каждый вызов assertEqual(Проверяет a == b), assertAlmostEqual(Проверяет a ≈ b (для float))
  
```
# Запуск тестов
 python3 -m unittest test_geometry.py

# Установка необходимое для тестирования с проверкой покрытия кода(coverage,pytest)
pip install coverage pytest pytest-cov

# Запуск тестов с проверкой покрытия
coverage run -m pytest

# Просмотр отчёта
coverage report -m
```
